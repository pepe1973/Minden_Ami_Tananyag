HTML - File elérési útvonal

  1. Nézzük a következő könyvtárstruktúrát.

    2. Abszolút file elérési útvonal:

  2. Abszolút URL (absolute URL): az erőforrás (resource) egy külső tárhelyen (nem része az alkalmazást tartalmazó könyvtárstruktúrának) található. Teljes elérési út (file path) megadása a gyökérelemtől, domain-től. Ebben az esetben például egy C: meghajtón lévő elemtől keressük a képeket.
    • F:/kulso_mappa/belso_mappa/kep_3.jpg
    • F:/kulso_mappa/kep_2.jpg
    • F:/kep_1.jpg
    • https://www.w3schools.com/images/img_girl.jpg

    3. Relatív file elérési útvonal:

  3. Relatív URL (relative URL): az adott állományt tartalmazó könyvtárstruktúrában van a hivatkozott erőforrás (resource).
  4. Két esetet vizsgálunk: szerverrel futtatjuk az állományt vagy File Managerből indítjuk. Mindig az index.html állományba írjuk a kódot. Nézzük először a következőt:
    • <img src="./kep_3.jpg" /> , ami pont ugyanaz, mint
    • <img src="kep_3.jpg" />
  5. Nézzük a következőt:
    • <img src="../kep_2.jpg" />
  6. Nézzük a következőt:
    • <img src="../../kep_1.jpg" />
  7. Viszont, ha következőt írjuk:
    • <img src="/kep_1.jpg" />
  8. Viszont, ha következőt írjuk:
    • <img src="/kep_2.jpg" />
  9. Viszont, ha következőt írjuk:
    • <img src="/kep_3.jpg" />
  10. Következtetésként a következőt vonhatjuk le. A szerverrel való indítás esetében a domain a belso_mappa, mert Visual Studio Code-ot kulso_mappa-ból indítottuk, így szerver csak "eddig" lát el, míg a második esetben a file://F, azaz a gyökérkönyvtár.
  11. Indítsuk a szervert (Visual Studio Code) belso_mappa-ból ekkor.
    • <img src="./kep_3.jpg" />
  12. Legyen
    • <img src="../kep_2.jpg" />
  13. Legyen
    • <img src="../../kep_1.jpg" />
  14. Nézzük a domain jelöléssel.
    • <img src="/kep_1.jpg" />
  15. Nézzük a domain jelöléssel.
    • <img src="/kep_2.jpg" />
  16. Nézzük a domain jelöléssel.
    • <img src="/kep_3.jpg" />
  17. Következtetésként a következőt vonhatjuk le. A szerverrel való indítás esetében a domain az index.html "környezete", mert Visual Studio Code-ot belso_mappa-ból indítottuk, így szerver csak "eddig" lát el, míg a második esetben a file://F, azaz a gyökérkönyvtár.
  18. Ezért mindig figyeljünk oda, hogy melyik mappából futtatjuk az alkalmazást!
