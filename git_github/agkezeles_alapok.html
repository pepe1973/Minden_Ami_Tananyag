Git és GitHub - Ágkezelés alapok

    Ágkezelés - Branching:

  1. Ág (branch): a fő vonaltól (main line) eltérő független fejlesztési vonal.
  2. Emlékeztetőként: a Git mindenegyes véglegesítéskor (commit) létrehoz egy pillanatfelvételt (snapshot) az állományrendszerről, a szerző nevével, e-mail címével és rövid üzenetével, a korábbi véglegesítésekre utaló mutatókkal (pointer), és hozzárendel egy ellenőrző összeget (checksum).

    Mindezt egy véglegesítési objektumban (commit object) tárolja el, amely tartalmaz egy mutatót (pointer) erre a pillanatfelvételre.

  3. Most részben a Git oldalán lévő példát is követjük. Ekkor a következő három állomány került véglegesítésre:

    git add README test.rb LICENSE

    git commit -m 'Initial commit'

    Véglegesítés után öt objektumom van.
    • Három blob (binary large object) objektumot, amelyek a három állományt tárolják.
    • Egy tree objektumot, amely ezt a három blob-ot.
    • És egy commit objektumot, amely erre a tree-re mutat, plusz néhány metadatot tartalmaz.
  4. Néhány véglegesítéssel később. Látható, hogy megjelent a parent tulajdonság is, amely a közvetlen elődre mutat.
  5. A Git-ben az ág (branch) egy mozgó mutató, amely általában az utolsó pillanatfelvételre mutat. Alapértelmezetten ez a master ág. Ezt a Git adja neki, de bármire át lehet nevezni. Például:

    git config --global init.defaultbranch main

  6. Hozzunk létre egy új ágat.

    git branch testing

    • Az utasítás hatására nem lépünk át automatikusan az új ágba!
    • Most már két mutatónk (master, testing) van.
  7. Nézzük meg naplót többféleképpen.
  8. Azt, hogy éppen melyik ágban vagyunk a HEAD pointer mutatja.
  9. Lépjünk át az új ágba. Ehhez ki kell adni a következő utasítást.

    git checkout testing

  10. Azt, hogy éppen melyik ágban vagyunk a HEAD pointer mutatja.
  11. Módosítsuk ebben az ágban a tartalmat egy véglegesítéssel. Naplozzuk ki a két ágat.
  12. Látható, hogy a két ág teljesen szétvált. A HEAD elem is folyamatosan az adott ágra mutat.

  13. git checkout master

  14. Az összes ág egyszeri naplózásához a következő utasítás kell

    git log --all

  15. Minden egyes ágváltáskor megváltozik a pillanatfelvételben lévő állományok rendszere (working directory)! Egyes állományok eltűnnek, mások megjelennek!
  16. Most változtassunk a master ágban.
  17. Grafikus formában az ágrajz a következő utasítással kapható meg.

    git log --oneline --decorate --graph --all

  18. Hasznos utasítás: ág létrehozása és átlépés egyszerre.

    git checkout -b ujAg

  19. Hasznos utasítások:
    • Átváltás ágak között.

      git switch master

    • Új ág létrehozása és átváltás ágak.

      git switch -c megujabbAg, vagy

      git switch --create megujabbAg

    • Visszaugrás az előző ágra.

      git switch -

    20. Források:

    git

    github
